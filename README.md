# *Frustum Fomula* : 바빌로니아와 이집트의 뿔대 부피 공식 분석 및 시뮬레이션

> **바빌로니아와 이집트 사람들은 어째서 뿔대의 부피를 다르게 구했는가 :**
> 공통수학1 다항식의 연산과 연계해 바빌로니아와 이집트의 두 뿔대 부피 공식을 비교하고 오차 근원을 알아본 심화 탐구 프로젝트입니다.

---

## 1. 탐구 동기
같은 도형의 부피를 구하는 공식에서 오차가 발생했다는 점에 주목해 그 이유를 탐색하고 싶었습니다. 특히 2D(사다리꼴)와 3D(뿔대)의 변화율 차이를 그래프로 시각화하면 공식으로 이루어진 부피 계산 과정을 직관적으로 확인하고 이해할 수 있을 것 같아, 수학 이해 보조 용도로 해당 Frustum Fomula 프로그램을 개발했습니다. 아래는 제가 궁금증을 느껴 심화해 탐구한 질문들입니다.
1. 두 공식의 오차 공식은 무엇인가
2. 항상 바빌로니아 공식이 이집트공식보다 같거나 더 큰 값을 도출하는 이유는?
3. 바빌로니아 공식의 오류는 왜 존재하는가
4. 이집트인들이 정확한 공식을 찾아낸 원리

## 2. 프로그램 주요 기능
- **부피 계산:** 밑면(a), 윗면(b), 높이(h) 입력 시 두 문명의 부피공식에 따라 계산결과를 즉시 비교
- **오차항 증명:** 바빌로니아 공식에서 이집트 공식을 뺀 오차항( (a-b)²/6 * h )과 실제 계산 오차의 일치 여부 확인
- **단면적 변화 시각화:** 높이에 따른 단면적의 변화(선형과 이차함수)를 Chart.js(차트 그려주는 자바스크립트 라이브러리입니다.)로 구현하여 오차의 실체를 시각적으로 제시

## 3. 심화 탐구 내용

### Q1. 바빌로니아 식은 왜 정확하지 않은가? -산술평균의 함정
- **탐색:** 바빌로니아는 단면적이 직선으로(일차함수) 줄어든다고 가정하여 **산술평균**(h(a²+b²)/2)을 사용했습니다.
- **결론:** 입체의 단면적은 길이의 제곱에 비례하여 **곡선**(이차함수)로 변화합니다. 직선은 항상 곡선(실제)보다 위쪽에 위치하게되어, 바빌로니아 식은 항상 **실제보다 큰 값**을 도출하게 됩니다. (a=b인 특수한 기둥형태 조건에는 오차항이 소거되어 바빌로니아 공식도 들어맞을 수 있습니다.)

### Q2. 이집트 공식은 어떻게 정밀함을 얻었는가?
- **탐색:** 이집트 식에 포함된 하인 ab는 두 밑면 넓이의 **기하평균**(a²*b²의 제곱근)입니다.
- **결론:** 닮음비를 통해 전체 뿔에서 작은 뿔을 빼는 과정을 거치면, 전체 뿔 높이가 소거되면서 ab라는 보정항이 나타납니다.
- 해당 이집트 공식은 정확한 이해를 위해 직접 계산해보았으나 마크다운 문서에 옮겨적는게 힘들어서 종이에 기록했습니다.

## 4. 탐구 결과 및 시사점
1. **변화율의 시각화:** 공식이 단순한 숫자의 나열이 아닌 그래프상에서 표현 가능한 추이로 확인 할 수 있다는 점 발견(사다리꼴과 뿔대의 비교를 통한 산술평균 오류 분석)
2. **수식의 분해와 의미:** ab와 같은 특정 항이 공식에 삽입된 이유를 분석, 각 항이 독립적인 역할을 가질 수 있다는 사실을 알았음.
3. **수학 이해를 위한 프로그래밍 도구 사용:** 공식에서 나타난 오차를 프로그래밍을 통해 시각화하여 분석했더니 확실히 이해에 도움이 됨.
